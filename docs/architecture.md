# Архітектура проєкту

## Основна ідея

Проєкт складається з кількох частин:

1. **Збір технічних даних** — Python-модулі виконують перевірки IP-адрес, доменів та ASN.
2. **Локальна база знань** — система використовує файли з папки `data/knowledge/` для додаткового пояснення результатів.
3. **AI-аналіз** — Gemini API формує підсумковий висновок на основі зібраних даних.


## Основні компоненти

### Streamlit UI

Файл `app.py` відповідає за вебінтерфейс користувача. Через нього користувач вводить IP-адресу, домен або ASN, запускає аналіз і бачить результати перевірок.

### Конфігурація

Модуль `core/config.py` відповідає за налаштування проєкту.

Він зчитує параметри з `.env`, наприклад:

- `GEMINI_API_KEY`;
- `GEMINI_MODEL`;
- таймаути;
- додаткові API-ключі, якщо вони використовуються.

Це дозволяє не зберігати секретні ключі напряму в коді.

### Перевірка введення

Модуль `core/security.py` перевіряє введені дані. Система приймає тільки:

- IP-адресу;
- домен;
- ASN, наприклад `AS15169`.

Якщо користувач вводить довільний текст або підозрілий запит, система не запускає аналіз.

### Структури даних

Модуль `core/schemas.py` містить структури даних, які використовуються між різними частинами проєкту.

Наприклад, там описуються об’єкти для:

- перевіреної цілі;
- результатів інструментів;
- підсумкового звіту.

Це потрібно, щоб дані передавались між модулями в однаковому форматі.

### Agent

Файл `core/agent.py` координує основну логіку роботи проєкту:

1. отримує перевірену ціль;
2. формує план перевірок;
3. запускає потрібні інструменти;
4. збирає результати в один звіт;
5. отримує додатковий контекст з локальної бази знань;
6. передає дані в Gemini API для підсумкового аналізу.

### Мережеві інструменти

У папці `tools/` розміщені модулі для технічних перевірок.

Основні перевірки поділені на два файли:

- `tools/network_tools.py` — локальні мережеві перевірки;
- `tools/external_intel.py` — отримання інформації із зовнішніх відкритих джерел.

У `tools/network_tools.py` реалізовано:

- ping-перевірку;
- traceroute / tracert;
- DNS lookup;
- reverse DNS lookup;
- перевірку популярних TCP-портів;
- отримання інформації про TLS-сертифікат.

У `tools/external_intel.py` реалізовано:

- GeoIP lookup;
- RDAP lookup;
- RIPEstat/BGP lookup;
- DNSBL-перевірку;
- опційну перевірку через AbuseIPDB, якщо налаштований API key.

### Локальна база знань

Файл `core/rag.py` працює з локальною базою знань у папці `data/knowledge/`.

У цій папці зберігаються Markdown-файли з короткими поясненнями про IP, DNS, TLS, BGP та безпеку. Система шукає релевантні фрагменти і додає їх до контексту AI-аналізу.

### Gemini API

Файл `core/gemini_client.py` відповідає за роботу з Gemini API.

Gemini використовується для двох задач:

1. вибір інструментів для аналізу, якщо увімкнений Gemini function calling;
2. формування підсумкового аналізу: пояснення результатів, можливих ризиків і рекомендацій.

Якщо `GEMINI_API_KEY` не вказаний або Gemini тимчасово недоступний, технічні перевірки все одно можуть виконуватись, але AI-висновок може бути замінений fallback-повідомленням.
